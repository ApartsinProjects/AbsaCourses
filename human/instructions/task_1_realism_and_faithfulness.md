# Task 1: Realism and Label Faithfulness

You will rate a set of student course reviews on two questions:

1. **Realism.** For every review, decide whether it was written by a real student or
   generated by a language model.
2. **Faithfulness.** For each review that turns out to be synthetic, check whether the
   pre-assigned aspect labels and sentiment polarities actually match what the review
   says.

The task is split into two parts so that your realism judgment in Part 1 is not biased
by seeing the synthetic labels.

You will rate the same items as the other raters but in independent random orders. Do
not discuss any item with another rater while Part 1 is open.

## Data

- Synthetic reviews are sampled from the project's main 10K/20-aspect generated corpus.
- Real reviews are sampled from the public Herath et al. 2022 student feedback corpus.
- Real and synthetic items are length-matched within reasonable tolerance so that you
  cannot trivially detect synthetic items by length alone.

## Workflow

1. Read the [codebook](../../codebook.md) once before you start.
2. Open your assigned Part 1 file and complete every row:
   `human/tasks/task_1_realism_and_faithfulness/rater_<LETTER>_part_1.csv`
3. Submit your completed Part 1 file before opening Part 2.
4. The coordinator releases the Part 2 file for you. Complete it.
5. Submit both completed files into `human/responses/task_1/`.

Expected time: Part 1 ~45-60 minutes, Part 2 ~30 minutes.

## Part 1: Realism judgment

Open `rater_<YOUR_LETTER>_part_1.csv`. Each row has:

| column        | meaning                                                       |
|---------------|---------------------------------------------------------------|
| `item_id`     | a stable identifier. Do not edit.                             |
| `review_text` | the review text. May be short, may be long. Do not edit.      |
| `judgment`    | **you fill**: `real` or `synthetic`.                          |
| `confidence`  | **you fill**: integer 1 through 5, defined below.             |
| `notes`       | optional. One line on what tipped you off, if anything.       |

Confidence scale:

| value | meaning                                       |
|-------|-----------------------------------------------|
| 1     | pure guess, could go either way               |
| 2     | slight lean, mostly guessing                  |
| 3     | moderate confidence                           |
| 4     | high confidence, would bet on this            |
| 5     | I am almost certain                           |

### What you are looking for

Use any cue you would normally use to distinguish real from synthetic text: vocabulary,
specificity (named TAs, assignments, lecture topics, tools), unevenness in argument
flow, partial contradictions, realistic frustration, sentence-level register, and
idiomatic phrasing.

Length alone is not a reliable cue here; the samples are roughly length-matched.

If you genuinely cannot decide, mark `judgment = synthetic` and `confidence = 1`, and
explain in `notes`. Do not leave the columns blank.

## Part 2: Label faithfulness

After you submit Part 1, the coordinator releases:
`human/tasks/task_1_realism_and_faithfulness/rater_<YOUR_LETTER>_part_2.csv`

This file contains only the synthetic items from Part 1, with their generator-assigned
aspect labels visible. One row per (review, aspect) pair.

| column               | meaning                                                                                    |
|----------------------|--------------------------------------------------------------------------------------------|
| `item_id`            | stable identifier. Do not edit.                                                            |
| `review_text`        | review text. Do not edit.                                                                  |
| `aspect`             | one of the 20 aspects (see [codebook](../../codebook.md)).                                 |
| `claimed_polarity`   | the polarity the synthetic generator assigned for this aspect.                             |
| `aspect_discussed`   | **you fill**: `yes`, `no`, or `unclear`.                                                   |
| `polarity_correct`   | **you fill**: `yes`, `no`, or `unclear`. Only meaningful when `aspect_discussed == yes`.   |
| `notes`              | optional. One line.                                                                        |

### How to decide

- `aspect_discussed = yes` when the review contains text addressing this aspect.
- `aspect_discussed = no` when the review does not address this aspect at all.
- `aspect_discussed = unclear` when there is a hint but it is too oblique to call.
- `polarity_correct = yes` when the claimed polarity is consistent with the text.
- `polarity_correct = no` when the text clearly conveys a different polarity.
- `polarity_correct = unclear` when the text is too vague to verify either way.
- If `aspect_discussed != yes`, you may leave `polarity_correct` as `unclear`.

Follow the codebook examples. Do not invent new aspect interpretations.
